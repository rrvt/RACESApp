RACES App Operations

Starting

When the app starts it loads the last database file known to the app. It then loads the drop down list in the dialog box with the current members information, sorted by name/call sign. At that point a member may be selected from the list or a search (find) may be used to display the fields of a single member. Alternately, one of the commands will load former memembers information into the drop down list. Then only former members may be displayed in the dialog box.

Making changes in the database is challenging. Make the wrong change and the database loses some of its purpose. So, the app is designed to make making changes inadvertently difficult. The status bar at the bottom of the screen shows the current mode of operation (Read Only or Save Record Changes). The inital mode of operation is Read Only. But Save Record Changes mode only allows the app to save record changes to the app's version of the database. In order to change the actual file which contains the database, the Update and Exit command must be executed and the dialog question responded to in the afirmative.

Load Database

The database is an .accdb file. A Open File dialog box starts and the user selects the correct file. The file is loaded into the app and the Drop Down List is loaded with current members. At that point the records are read only and the status bar describes the path and the content of the drop down list of members.

Create New Member

The drop down list is cleared, the fields are cleared and those fields with an explanation have labels stored in them. Labels are distintuished from data by angle bracket, "<label>". The mode is changed to Save Records (in liu of Read Only). Filling in the fields is only the first step. Then the records need to be saved which happens automatically when any other action is executed. Then the update and exit command needs to be executed with an affirmative response to the update question.

Load Current Members

Clear the drop down list and load the current members. Reset the status to reflect the new list.

Load Former Members

Clear the drop down list and load the former members. Reset the status to reflect the new list.

Edit Records

Change the mode from Read Only to Save Records. There is no command to simply return to Read Only mode.

Open Member's Check List

Opens a dialog box for the member displayed that has a check list of all the things that should be done for a new member (or maybe a member being transferred to the former member status). The results of the check list are kept in the ini file.

Remove Database Dead Records

Each member record contains identifiers (integers) of Entitiy records. Each Entity Record contains two indentifiers (for an address and city record). When a member record is deleted up to nine records may have been orphaned. However, some of them may be used in other member or entity records and cannot be deleted. This command verifies every record is used somewhere otherwise it is marked to be deleted.

About

Displays the About Box.

Update Database and Exit

If any records have been modified and saved in the apps version of the database and dialog box is displayed asking if the database should be updated. if the answer is yes a backup file is created before the file is updated. Then the app exits.
